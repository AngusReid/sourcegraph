{{template "h1-id" dict "ID" "introduction" "Name" "API Introduction"}}

Before getting started, it's important to understand the underlying concepts at hand.

{{template "h2-id" dict "ID" "grpc" "Name" "gRPC"}}

Sourcegraph uses a high-performance RPC framework called gRPC to provide API access, this means:

  • Generated libraries for over over ten languages can be used to access the Sourcegraph API!
  • The entire API is described generic and simple Protobuf files.

For more information about gRPC check out the gRPC FAQ.

{{template "h2-id" dict "ID" "restful" "Name" "RESTful (coming soon)"}}

Unfortunately gRPC cannot yet be used in browser-based applications due to technical limitations. For this reason the Sourcegraph API will also be available as a first-class HTTP/1.1 RESTful JSON API.

Note: The Sourcegraph RESTful API is being actively worked on and will be available soon! Follow us on Twitter to get the latest news.

{{template "h2-id" dict "ID" "structure" "Name" "Structure"}}

The usage of two different APIs (gRPC and REST) means that in the documentation is structures as such:

  • Broken into services, messages, and enumerations (these are gRPC/Protobuf concepts).
  • How these gRPC concepts apply to the REST API (the HTTP URL, method, it's fields, etc).

Remember that you can always view documentation specific to your languages API library (i.e. code rather than concepts) by clicking on the documentation link provided on the language-specific library.

Once you get started, you'll see that the documentation is very nice! To dive in just click on a service that sounds interesting on the right!
